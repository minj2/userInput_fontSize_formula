UX & Mobile Optimization Analysis

Large text (56–99px) — Aggressive scaling

Hero headings and banners scale down to 38–48px on mobile. Prevents horizontal overflow and maintains visual hierarchy without manual breakpoints.

Medium text (24–55px) — Gentle scaling

Section headings shrink proportionally. The curve formula 0.23x + 25 ensures the reduction feels natural — not jarring — across viewport sizes.

Small text (≤24px) — No scaling, 1:1

Body text, captions, and labels stay exactly the same size on mobile and desktop. Minimum readability is always preserved — no media query needed.

Fluid middle zone — vw-based

Between mobile (375px) and desktop (1440px), font size scales fluidly with viewport width. No abrupt jumps at breakpoints — smooth transition throughout.

User control fully respected

Desktop always renders the exact px value the user typed. The formula only adjusts mobile — users get predictable results on the screen they're designing for.

Per-tag independence — no hierarchy collisions

Each bb-font tag gets the formula applied independently. No ratio-based cascading means body text can never accidentally exceed h1 size on mobile.
